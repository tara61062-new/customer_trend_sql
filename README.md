📊 Project Overview

**Customer Shopping Trend Analysis** is a SQL-based data analytics project that explores customer purchasing behavior, product performance, subscription patterns, and revenue trends using PostgreSQL.

The project leverages SQL queries to transform transactional customer data into meaningful business insights. By analyzing demographics, spending habits, discounts, subscriptions, shipping preferences, and customer loyalty, the project demonstrates how SQL can be used to support data-driven business decisions.

## Business Questions Solved

### Revenue Analysis

1. What is the total revenue generated by male vs female customers?
2. What is the revenue contribution of each age group?

### Customer Spending Behavior

3. Which customers used a discount but still spent more than the average purchase amount?
4. Do subscribed customers spend more? Compare average spend and total revenue between subscribed and non-subscribed customers.
5. Are customers who are repeat buyers (more than 5 previous purchases) also likely to subscribe?

### Product Performance

6. Which are the top 5 products with the highest average review rating?
7. Which 5 products have the highest percentage of purchases with discounts applied?
8. What are the top 3 most purchased products within each category?

### Shipping Analysis

9. Compare the average purchase amounts between Standard and Express shipping methods.

### Customer Segmentation

10. Segment customers into:

* New Customers
* Returning Customers
* Loyal Customers

and determine the count of customers in each segment.

## How to Run

### 1. Create a PostgreSQL Database

```sql
CREATE DATABASE customer_shopping_trend;
```

### 2. Import Dataset

Import the `customer_shopping_behavior.csv` file into PostgreSQL using pgAdmin or the `COPY` command.

### 3. Execute SQL Queries

Run the queries from:

```text
customer_shopping_trend_analysis.sql
```

### 4. Review Results

Analyze the query outputs to understand customer behavior, revenue trends, and product performance.
